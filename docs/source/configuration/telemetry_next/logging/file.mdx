title: Logging in the Apollo Router
---

## Logging to Files

To log to a file enable the option in your `router.yaml`:

```yaml title="router.yaml"
telemetry:
  logging:
    file:
      enabled: true
```

By default, the router will log to `router.log` in the current working directory.

### Configuration

#### `enabled`

To enable file logging set the `enabled` option to `true`:

```yaml title="router.yaml"
telemetry:
  logging: 
    file:
      enabled: true
```

#### `format`

File logging can be configured to use a variety of different formats. The default format is `text`.

```yaml title="router.yaml"
telemetry:
  logging: 
    file:
      enabled: true
      format: text
```

See the [format documentation](format.mdx) for more information.

#### `path`
By default, the Router will write to `router.log` in the current working directory.

To change the location of the log file set the `path` option:
```yaml title="router.yaml"
telemetry:
  logging: 
    file:
      enabled: true
      path: /tmp/router.log
```

#### `rollover`

The router can be configured to roll over to a different file name depending on the `rollover` interval.

The log file pattern is as follows:
* `hourly` - `<path>.YYYY-MM-DD-HH`
* `daily` - `<path>.YYYY-MM-DD`
* `never` - `<path>`

Setting the rollover to anything other than `never` will cause the Router to write to a new file at the specified interval with the file  

```yaml title="router.yaml"
telemetry:
  logging: 
    file:
      enabled: true
      rollover: daily
```

### Configuration reference
| Option                | Values                     | Default      | Description                                           |
|-----------------------|----------------------------|--------------|-------------------------------------------------------|
| `enabled`             | `true`\|`false`            | `false`      | Enable or disable file logging.                       |
| `format`              |                            | `text`       | See the [format documentation](#format) for details.  |
| `path`                |                            | `router.log` | The file system location to log to.                   |
| `rollover`            | `hourly`\|`daily`\|`never` | `never`      | The frequency to rollover.                            |
